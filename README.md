# T9 Wizard

A "Typing of the Dead"-style typing-combat game built for KaiOS feature phones — instead of a QWERTY keyboard, you fight off incoming words using the T9 numeric keypad (2-9).

Colored blocks carrying a word fall from the top of the screen toward a grey player block at the bottom. Type the T9 digit sequence for a word before its block reaches the player to destroy it. Let one through and you lose a life; lose all 3 and it's game over.

## How it plays

- **Canvas**: hard-coded 240x320. The top 294px is the play field (enemies fall, player sits at the bottom); the bottom 26px is a HUD strip (lives, level/kill counter).
- **T9 typing**: words are typed by their numeric-keypad encoding (2=ABC, 3=DEF, 4=GHI, 5=JKL, 6=MNO, 7=PQRS, 8=TUV, 9=WXYZ). E.g. "butterfly" → `288837359`.
- **Targeting (lock-on)**: pressing a digit locks onto an enemy whose word starts with it. If more than one enemy matches, the one closest to the player (i.e. furthest fallen) is locked first. Once locked, that target **cannot** be changed or abandoned — wrong digits are simply ignored (no penalty, no reset) until the locked word is fully typed and the enemy dies.
- **Lives**: 3 lives, shown as red squares in the bottom-left of the HUD. An enemy reaching the player costs one life; 0 lives ends the game.
- **Levels**: defeating 20 enemies advances the level. Word length scales with level via a shifting 3-length window, capped at 12 chars so words always fit on screen (level 1 = 2-4 letters, level 2 = 3-5, ... level 9+ caps at 10-12). See `getWordLengthRangeForLevel` in `js/game.js`.
- **Word list**: a curated, filtered list of common conversational English words (not the full dictionary) — see [Word data](#word-data) below.

## Project structure

```
index.html            Canvas + script tags (load order matters)
manifest.webapp        Minimal KaiOS app manifest
css/style.css           Canvas scaling (matches viewport width, keeps aspect ratio)
js/
  t9.js                 Letter -> T9 digit map + wordToT9Code()
  words.js              WordBank: picks a random word for a given length range
  enemy.js               Enemy data factory
  input.js                InputEngine: the lock-on/typing state machine (core mechanic)
  render.js                All canvas drawing (play field, enemies, HUD, overlays)
  game.js                   State machine, game loop, spawning, collisions, level-up
  main.js                    Bootstrap: wires up keydown handling and starts the game
data/
  words-data.js          Hard-coded word-by-length data (`WORDS_BY_LENGTH`), loaded via
                         a plain <script> tag — no fetch(), no build step, works from file://
icons/                 Placeholder app icons (solid color, for the KaiOS manifest)
release.sh             Uploads all .js/.html/.webapp/.css files to S3 (see script for bucket/prefix)
```

## Running it

No build step, no dependencies, no server required — just open `index.html` directly in a browser (`file://` works fine, since the word data is a plain script include rather than a fetch()'d JSON file).

For on-device KaiOS testing, sideload the directory as a packaged app using `manifest.webapp`.

Controls: digit keys **2-9** to type. Any digit on the menu or game-over screen starts/restarts the game.

## Word data

`data/words-data.js` is hand-maintained (not regenerated by a build script — the original generation pipeline and source dictionary were removed once the list was finalized). It started from a frequency-ranked common-English-words list, filtered to drop acronyms/abbreviations/brand names, patched for a few common words a dictionary cross-check had incorrectly dropped, and then hand-reviewed to strip profanity, slurs, sexual-content terms, and a few sensitive-topic words (e.g. violent/traumatic terms) that aren't a good fit for a general-audience game. If you need to add or remove words, edit the arrays in `data/words-data.js` directly.

## Tuning knobs

Game feel (spawn rate, fall speed, colors, level word-length formula) all live as named constants at the top of `js/game.js` — e.g. `SPAWN_INTERVAL_MS`, `ENEMY_FALL_SPEED`, `ENEMY_SPEED_JITTER`, `KILLS_PER_LEVEL`, `COLOR_PALETTE`. Currently only word length scales with level; fall speed/spawn rate are intentionally flat across levels until tuned otherwise.
